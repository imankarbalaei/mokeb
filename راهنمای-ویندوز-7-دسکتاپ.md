# راهنمای اجرای نسخه دسکتاپ روی Windows 7

این راهنما مراحل راه‌اندازی پروژه «موکب الشهدا قم» را از صفر روی Windows 7 توضیح می‌دهد؛ از نصب Git و Node.js تا کلون گرفتن پروژه، نصب وابستگی‌ها، اجرای برنامه دسکتاپ و ساخت فایل اجرایی `exe`.

فرض این راهنما:

- سیستم مقصد Windows 7 است.
- خروجی پیشنهادی ۳۲ بیتی است.
- برنامه قرار است به‌صورت Desktop App اجرا شود، نه داخل مرورگر.

## ۱. نصب Git

ابتدا Git for Windows را نصب کنید.

برای Windows 7 ممکن است آخرین نسخه Git نصب نشود. اگر نصب آخرین نسخه خطا داد، از نسخه‌های قدیمی‌تر Git for Windows استفاده کنید که با Windows 7 سازگار هستند.

بعد از نصب، از Start Menu برنامه زیر را باز کنید:

```text
Git Bash
```

برای تست نصب Git این دستور را بزنید:

```bash
git --version
```

اگر شماره نسخه نمایش داده شد، Git درست نصب شده است.

## ۲. نصب Node.js مناسب Windows 7

برای Windows 7 از نسخه‌های جدید Node.js استفاده نکنید.

نسخه پیشنهادی:

```text
Node.js 16.x
```

نسخه‌های جدیدتر Node.js ممکن است روی Windows 7 نصب یا اجرا نشوند.

بعد از نصب Node.js، بهتر است یک بار ویندوز را Restart کنید.

سپس Git Bash یا Command Prompt را باز کنید و این دستورها را بزنید:

```bash
node -v
npm -v
```

خروجی مناسب باید شبیه این باشد:

```text
v16.x.x
8.x.x
```

اگر دستور `node` یا `npm` شناخته نشد، نصب Node.js کامل نشده یا مسیر آن به PATH اضافه نشده است.

## ۳. ساخت پوشه پروژه

یک مسیر ساده برای پروژه بسازید، مثلاً:

```text
C:\projects
```

بعد Git Bash را باز کنید و وارد این مسیر شوید:

```bash
cd /c/projects
```

## ۴. کلون گرفتن پروژه از GitHub

برای دریافت پروژه:

```bash
git clone https://github.com/imankarbalaei/mokeb.git
```

بعد وارد پوشه پروژه شوید:

```bash
cd mokeb
```

## ۵. نصب وابستگی‌های پروژه

داخل پوشه پروژه این دستور را اجرا کنید:

```bash
npm install
```

این مرحله به اینترنت نیاز دارد و ممکن است چند دقیقه طول بکشد.

اگر در این مرحله خطا گرفتید، اول نسخه Node.js را بررسی کنید:

```bash
node -v
```

برای Windows 7 بهتر است نسخه Node.js همان 16 باشد.

## ۶. اجرای برنامه به‌صورت Desktop برای تست

بعد از نصب وابستگی‌ها، برای اجرای برنامه به‌صورت پنجره دسکتاپ:

```bash
npm run desktop
```

این دستور:

1. نسخه نهایی رابط کاربری را می‌سازد.
2. برنامه را داخل پنجره دسکتاپ Electron باز می‌کند.

در این حالت لازم نیست آدرس مرورگر باز کنید.

## ۷. ساخت فایل اجرایی Portable برای Windows 7 سی‌ودو بیتی

برای ساخت فایل اجرایی `exe` سی‌ودو بیتی:

```bash
npm run dist:win32
```

بعد از اتمام، خروجی داخل پوشه زیر ساخته می‌شود:

```text
release
```

فایل اصلی خروجی معمولاً نامی شبیه این دارد:

```text
Mokeb-Al-Shohada-Qom-1.0.0-win-ia32.exe
```

این فایل Portable است و معمولاً بدون نصب اجرا می‌شود.

## ۸. اجرای برنامه نهایی

داخل پوشه `release` روی فایل زیر دوبار کلیک کنید:

```text
Mokeb-Al-Shohada-Qom-1.0.0-win-ia32.exe
```

برنامه باید مثل یک نرم‌افزار دسکتاپ ویندوزی باز شود.

## ۹. اگر سیستم ۶۴ بیتی بود

اگر مطمئن شدید سیستم مقصد Windows 7 شصت‌وچهار بیتی است، می‌توانید خروجی ۶۴ بیتی بسازید:

```bash
npm run dist:win64
```

اما برای Windows 7، خروجی ۳۲ بیتی معمولاً انتخاب امن‌تری است، چون روی بسیاری از سیستم‌های ۳۲ و ۶۴ بیتی اجرا می‌شود.

## ۱۰. تغییر دستی بیت خروجی در package.json

داخل فایل `package.json` این scriptها وجود دارند:

```json
"dist:win32": "npm run build && electron-builder --win --ia32",
"dist:win64": "npm run build && electron-builder --win --x64"
```

معنی مقادیر:

- `--ia32` یعنی خروجی ۳۲ بیتی
- `--x64` یعنی خروجی ۶۴ بیتی

اگر خواستید دستور پیش‌فرض ۳۲ بیتی را به ۶۴ بیتی تبدیل کنید، مقدار `--ia32` را به `--x64` تغییر دهید.

## ۱۱. نکته مهم درباره اطلاعات برنامه

اطلاعات ورود، خروج و وضعیت جایگاه‌ها روی همان سیستم ذخیره می‌شود.

یعنی:

- با بستن و باز کردن برنامه اطلاعات باقی می‌ماند.
- اطلاعات بین چند سیستم مشترک نیست.
- اگر داده‌های برنامه یا کش کاربر پاک شود، اطلاعات ممکن است حذف شود.

برای استفاده عملیاتی، بهتر است همیشه از یک سیستم ثابت استفاده شود.

## ۱۲. خطاهای رایج

### دستور git شناخته نمی‌شود

Git نصب نیست یا Git Bash را باز نکرده‌اید.

راه‌حل:

- Git for Windows را نصب کنید.
- از برنامه Git Bash استفاده کنید.

### دستور node یا npm شناخته نمی‌شود

Node.js نصب نیست یا PATH درست تنظیم نشده است.

راه‌حل:

- Node.js 16 را نصب کنید.
- ویندوز را Restart کنید.
- دوباره `node -v` و `npm -v` را تست کنید.

### npm install خطا می‌دهد

اول نسخه Node.js را چک کنید:

```bash
node -v
```

اگر نسخه خیلی جدید است، آن را حذف کنید و Node.js 16 نصب کنید.

### برنامه روی Windows 7 باز نمی‌شود

چند مورد را بررسی کنید:

- خروجی ۳۲ بیتی ساخته باشید: `npm run dist:win32`
- Windows 7 آپدیت‌های پایه را داشته باشد.
- آنتی‌ویروس فایل exe را مسدود نکرده باشد.
- فایل را از مسیر خیلی طولانی یا پوشه فارسی/خاص اجرا نکنید؛ مثلاً آن را داخل `C:\mokeb` تست کنید.

## ۱۳. خلاصه دستورها

از صفر تا اجرای تستی:

```bash
git clone https://github.com/imankarbalaei/mokeb.git
cd mokeb
npm install
npm run desktop
```

برای ساخت فایل اجرایی ۳۲ بیتی:

```bash
npm run dist:win32
```

برای ساخت فایل اجرایی ۶۴ بیتی:

```bash
npm run dist:win64
```
