# راهنمای نصب و استفاده از برنامه مدیریت موکب الشهدا قم

این سند مراحل نصب، اجرا و استفاده از برنامه مدیریت استراحتگاه «موکب الشهدا قم» را روی ویندوز توضیح می‌دهد.

برنامه دو حالت دارد:

- حالت وب برای توسعه و تست
- حالت دسکتاپ ویندوز با Electron، مناسب زمانی که کاربر نخواهد برنامه را داخل مرورگر باز کند

## ۱. پیش‌نیازها

برای اجرای برنامه روی ویندوز، این موارد لازم است:

- ویندوز ۱۰ یا ۱۱
- مرورگر به‌روز مثل Chrome، Edge یا Firefox
- نصب بودن Node.js

برای ویندوز ۷ باید از نسخه دسکتاپ ساخته‌شده با Electron 22 استفاده شود. نسخه‌های جدیدتر Electron از ویندوز ۷ پشتیبانی نمی‌کنند.

## ۲. نصب Node.js روی ویندوز

۱. وارد سایت رسمی Node.js شوید:

```text
https://nodejs.org
```

۲. نسخه LTS را دانلود کنید.

۳. فایل نصب را اجرا کنید.

۴. در مراحل نصب، گزینه‌های پیش‌فرض را تغییر ندهید و نصب را کامل کنید.

۵. بعد از نصب، یک پنجره Command Prompt یا PowerShell باز کنید و این دستورها را بزنید:

```bash
node -v
npm -v
```

اگر شماره نسخه نمایش داده شد، نصب درست انجام شده است.

## ۳. آماده‌سازی برنامه

۱. پوشه پروژه را روی سیستم ویندوز کپی کنید.

مثلاً:

```text
C:\mokeb
```

۲. داخل همان پوشه، در فضای خالی راست‌کلیک کنید و گزینه Terminal یا Open in Terminal را بزنید.

اگر این گزینه را ندارید، PowerShell را باز کنید و با دستور زیر وارد پوشه شوید:

```powershell
cd C:\mokeb
```

۳. وابستگی‌های برنامه را نصب کنید:

```bash
npm install
```

این دستور فقط بار اول لازم است. ممکن است چند دقیقه طول بکشد.

## ۴. اجرای برنامه برای استفاده روزانه

برای اجرای برنامه، داخل پوشه پروژه این دستور را بزنید:

```bash
npm run dev
```

بعد از اجرا، آدرسی شبیه این نمایش داده می‌شود:

```text
http://localhost:5173
```

این آدرس را در مرورگر باز کنید.

نکته: تا وقتی برنامه در حال استفاده است، پنجره Terminal را نبندید.

## ۵. ساخت نسخه نهایی

اگر خواستید نسخه نهایی برنامه را بسازید، این دستور را اجرا کنید:

```bash
npm run build
```

بعد از اجرای موفق، پوشه‌ای به نام `dist` ساخته می‌شود. این پوشه نسخه نهایی برنامه است.

برای مشاهده نسخه نهایی روی همان سیستم:

```bash
npm run preview
```

سپس آدرس نمایش‌داده‌شده را در مرورگر باز کنید.

## ۵.۱. اجرای برنامه به‌صورت دسکتاپ

اگر می‌خواهید برنامه مثل یک نرم‌افزار ویندوزی باز شود و نه داخل مرورگر، از حالت دسکتاپ استفاده کنید.

برای تست روی سیستم توسعه:

```bash
npm run desktop
```

این دستور ابتدا نسخه نهایی وب را می‌سازد و بعد آن را داخل پنجره دسکتاپ Electron باز می‌کند.

## ۵.۲. ساخت فایل اجرایی برای Windows 7 سی‌ودو بیتی

دستور پیش‌فرض پیشنهادی برای Windows 7 خروجی سی‌ودو بیتی می‌سازد.

برای ساخت خروجی:

```bash
npm run dist:win32
```

بعد از اتمام، خروجی‌ها داخل پوشه زیر قرار می‌گیرند:

```text
release
```

خروجی اصلی یک فایل Portable است:

- نسخه Portable بدون نصب اجرا می‌شود.
- برای ویندوز ۷ معمولاً همین گزینه ساده‌تر و کم‌دردسرتر است.

اگر واقعاً Installer لازم داشتید و روی ویندوز build می‌گیرید، از دستور زیر استفاده کنید:

```bash
npm run dist:win32:installer
```

برای Installer شصت‌وچهار بیتی:

```bash
npm run dist:win64:installer
```

نکته: ساخت Installer ویندوز روی Linux معمولاً به Wine نیاز دارد. برای همین خروجی پیش‌فرض پروژه Portable است.

## ۵.۳. تغییر خروجی از ۳۲ بیت به ۶۴ بیت

اگر بعداً مطمئن شدید ویندوز مقصد ۶۴ بیتی است، ساده‌ترین راه استفاده از دستور آماده زیر است:

```bash
npm run dist:win64
```

اگر خواستید دستور پیش‌فرض ۳۲ بیتی را به ۶۴ بیتی تبدیل کنید، فایل `package.json` را باز کنید و در بخش `scripts` این خط را پیدا کنید:

```json
"dist:win32": "npm run build && electron-builder --win --ia32"
```

و `--ia32` را به `--x64` تغییر دهید:

```json
"dist:win32": "npm run build && electron-builder --win --x64"
```

توضیح:

- `--ia32` یعنی خروجی ۳۲ بیتی
- `--x64` یعنی خروجی ۶۴ بیتی

برای سازگاری بیشتر با ویندوز ۷های قدیمی، خروجی ۳۲ بیتی انتخاب امن‌تری است؛ حتی روی بسیاری از ویندوزهای ۶۴ بیتی هم برنامه ۳۲ بیتی اجرا می‌شود.

## ۵.۴. نکته مهم درباره ساخت exe ویندوز

بهترین حالت این است که فایل exe ویندوز روی خود ویندوز ساخته شود.

روی Linux یا macOS هم ممکن است بتوان خروجی ویندوز گرفت، اما برای installer ویندوز معمولاً ابزارهای اضافه مثل Wine لازم می‌شود و احتمال خطا بیشتر است.

پیشنهاد عملی:

- برای ساخت نسخه نهایی ویندوز، پروژه را ترجیحاً روی ویندوز ۱۰ یا ۱۱ آماده کنید.
- اگر مجبورید روی ویندوز ۷ build بگیرید، از Node.js 16 استفاده کنید؛ نسخه‌های جدید Node ممکن است روی ویندوز ۷ درست نصب یا اجرا نشوند.
- داخل پوشه پروژه `npm install` را اجرا کنید.
- سپس `npm run dist:win32` را بزنید.

برای اجرای فایل خروجی روی سیستم مقصد، کاربر نهایی نیازی به نصب Node.js ندارد.

## ۶. روش کار با برنامه

### داشبورد اصلی

در بالای صفحه عنوان «موکب الشهدا قم» نمایش داده می‌شود.

داشبورد سه آمار اصلی دارد:

- کل جایگاه‌ها: ۵۴۰
- جایگاه‌های اشغال‌شده
- جایگاه‌های خالی

جدول جایگاه‌ها شامل ردیف‌های A تا I است و هر ردیف ۶۰ جایگاه دارد.

رنگ‌ها:

- سبز: جایگاه خالی
- قرمز: جایگاه اشغال‌شده

## ۷. ثبت ورود زائر

۱. روی یک جایگاه سبز کلیک کنید.

۲. فرم ورود باز می‌شود.

۳. اطلاعات زیر را وارد کنید:

- نام و نام خانوادگی
- شماره تلفن
- تاریخ ورود
- ساعت ورود

۴. روی دکمه «ثبت ورود» کلیک کنید.

بعد از ثبت، جایگاه قرمز می‌شود و وضعیت آن به اشغال‌شده تغییر می‌کند.

## ۸. ثبت خروج زائر

۱. روی یک جایگاه قرمز کلیک کنید.

۲. اطلاعات زائر نمایش داده می‌شود.

۳. روی دکمه «ثبت خروج و آزادسازی» کلیک کنید.

برنامه زمان خروج را ثبت می‌کند و جایگاه دوباره سبز و قابل استفاده می‌شود.

## ۹. چاپ لیبل

برای هر جایگاه اشغال‌شده امکان چاپ لیبل وجود دارد.

۱. روی جایگاه قرمز کلیک کنید.

۲. در پنجره بازشده، پیش‌نمایش لیبل را ببینید.

۳. روی دکمه «چاپ لیبل» کلیک کنید.

لیبل برای چاپگر Godex G500 و لیبل‌های کوچک ۳۴×۵۱ میلی‌متر آماده شده است. اندازه کاغذ از داخل برنامه به چاپگر تحمیل نمی‌شود؛ بهتر است سایز لیبل در تنظیمات خود درایور/پرینتر تعریف شود.

اطلاعات روی لیبل:

- عنوان «موکب الشهدا قم»
- بارکد/QR نمایشی
- نام زائر
- ردیف و شماره جایگاه
- زمان ورود

### تنظیمات پیشنهادی چاپ

در پنجره چاپ مرورگر:

- اندازه کاغذ را در تنظیمات خود درایور Godex روی لیبل 34 × 51 mm یا preset درست دستگاه قرار دهید.
- Margins یا حاشیه را روی None یا Minimum بگذارید.
- Scale را روی 100% قرار دهید.
- اگر گزینه Headers and Footers فعال است، آن را غیرفعال کنید.

## ۱۰. ذخیره اطلاعات

اطلاعات برنامه در مرورگر همان سیستم ذخیره می‌شود.

یعنی:

- با Refresh کردن صفحه اطلاعات حذف نمی‌شود.
- اگر مرورگر عوض شود، اطلاعات قبلی دیده نمی‌شود.
- اگر Cache یا Site Data مرورگر پاک شود، اطلاعات برنامه هم ممکن است حذف شود.

پیشنهاد عملی: برای استفاده روزانه، همیشه از یک مرورگر و یک سیستم ثابت استفاده کنید.

## ۱۱. جستجو

در بالای جدول جایگاه‌ها یک کادر جستجو وجود دارد.

می‌توانید بر اساس موارد زیر جستجو کنید:

- شماره جایگاه مثل `B-14`
- نام زائر
- شماره تلفن

## ۱۲. نکات مهم عملیاتی

- قبل از بستن برنامه، مطمئن شوید عملیات ورود یا خروج کامل ثبت شده است.
- برای جلوگیری از از دست رفتن داده‌ها، Site Data مرورگر را پاک نکنید.
- اگر چند سیستم هم‌زمان استفاده شوند، اطلاعات بین آن‌ها مشترک نیست؛ این برنامه فعلاً محلی و مبتنی بر مرورگر است.
- برای استفاده چندکاربره یا شبکه‌ای، باید نسخه سرور/دیتابیس‌دار جداگانه توسعه داده شود.

## ۱۳. رفع اشکال‌های رایج

### دستور npm شناخته نمی‌شود

Node.js نصب نیست یا نصب کامل نشده است. Node.js نسخه LTS را نصب کنید و سیستم را یک بار Restart کنید.

### صفحه برنامه باز نمی‌شود

بررسی کنید دستور زیر هنوز در Terminal در حال اجرا باشد:

```bash
npm run dev
```

سپس آدرس نمایش‌داده‌شده را دوباره در مرورگر باز کنید.

### اطلاعات بعد از باز کردن در مرورگر دیگر دیده نمی‌شود

طبیعی است. اطلاعات در localStorage همان مرورگر ذخیره می‌شود.

### چاپ لیبل اندازه درست ندارد

در تنظیمات چاپ و درایور Godex، سایز لیبل 34 × 51 mm، مقدار Scale و Margin را بررسی کنید. Scale باید 100% و حاشیه‌ها باید حذف یا حداقلی باشند.
